屏幕监测器 (ScreenMonitor)，使用deepseek编写。  
实时监测屏幕指定区域，当画面与预设一致时自动发出系统提示音。适用于游戏自动化、状态监控、辅助操作等场景。

功能特点  
📸 屏幕截图检测：基于辅助功能截图 API，无需 Root，最低要求 Android 11 (API 30)。

🎯 自定义监测区域：从相册选择一张与手机分辨率相同的截图，手指框选需要监测的区域。

⚙️ 可调参数：支持动态调整相似度阈值、颜色容差、检测间隔。

🔊 声音提醒：匹配成功时播放系统默认通知音。

📋 内置日志：可在应用内实时查看运行日志，方便调试。

🔔 通知栏状态：可开关的通知栏提示，显示当前相似度或匹配状态。

使用步骤  
准备预设图  
对需要监测的界面截一张图（分辨率需与手机屏幕完全一致），保存到相册。

框选区域  
打开应用 → 点击 “从相册选择预设图片” → 选中刚才的截图 → 在图片上用手指框选需要监测的矩形区域 → 点击 “确认框选区域”。

配置参数（主界面）

相似度阈值：建议 0.80 ~ 0.95，值越高匹配越严格。

颜色容差：数值越大颜色偏差容忍度越高，复杂画面可适当增大。

监测间隔：秒数，最短 1 秒。

开启无障碍服务  
点击确认后会自动跳转无障碍设置页，找到 “屏幕监测器”（或 ScreenMonitor）并开启服务。  
若已开启，则直接返回主界面。

开始监测  
服务启动后，通知栏会出现常驻通知（可在主界面关闭）。当手机显示的画面中，被框选区域与预设图的相似度超过阈值时，手机会发出提示音，通知栏显示 “匹配成功！”。

查看日志  
点击主界面 “查看日志” 可实时观察服务运行状态、截图结果、相似度数值等。

技术实现  
辅助功能截图：AccessibilityService.takeScreenshot() (API 30+)

像素对比优化：逐行批量读取像素，颜色容差算法，硬件位图兼容

前台服务通知：使用普通通知避免 FOREGROUND_SERVICE 权限

权限管理：Android 13+ 动态请求 POST_NOTIFICATIONS 权限
